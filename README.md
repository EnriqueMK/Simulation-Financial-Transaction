# 🏦 Simulation Transaction Finance

O **Simulation Transaction Finance** é um sistema desenvolvido em Java, baseado nos princípios de Programação Orientada a Objetos (POO), com o objetivo de simular transações financeiras realizadas via cartão de crédito.

A aplicação permite o cadastro de clientes, emissão e vinculação de cartões, controle de limite e processamento de transações, armazenando todas as informações geradas durante as operações.

---

<h2>🎯 Objetivo do Projeto</h2>
Este projeto foi desenvolvido com foco em:

Aplicação prática de conceitos de POO

- Modelagem de entidades financeiras
- Controle de regras de negócio
- Simulação de fluxo transacional
- Organização profissional de versionamento com Git

---

<h2>🧱 Conceitos Aplicados</h2>

---

<h2>🌿 Estrutura de Branches</h2>
O projeto utiliza uma organização de branches para facilitar o monitoramento, a rastreabilidade, a manutenção e a correção do código.

**main** → Versão estável do sistema  
**feature/programar-models** → Desenvolvimento de novas funcionalidades nos models  
**feature/programar-validation** → Desenvolvimento de validações e serviços para garantir o correto funcionamento dos models  
**fix/bugs-models** → Correção de bugs identificados durante o desenvolvimento dos models  
**fix/bugs-validation** → Correção de bugs identificados durante o desenvolvimento das validações  
**chore/project-organization** → Organização e manutenção do projeto

<h2>🧠 Estrutura Conceitual do Sistema</h2>

O sistema é estruturado com separação clara entre:

- **Models** → Representação das entidades (Cliente, Cartão, Transação)
- **Validações/Service**s → Regras de negócio e verificações
- **Camada de execução** → Simulação do fluxo principal

Essa divisão permite:

- Maior manutenibilidade
- Melhor testabilidade
- Facilidade de expansão futura
